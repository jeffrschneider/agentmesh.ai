The web ran on a stack of protocols, each with one job: find, reach, trust, talk. The agent web is assembling the same stack. AgentMesh is its connective layer: how any agent reaches any other, trusts what it hears, and knows who's there right now.

the stack

The web took a stack. So does this.

The web was never one protocol: DNS found, TCP reached, TLS trusted, HTTP talked. The agent web is assembling the same kind of stack, one spec per job. AgentMesh claims the connective jobs: the ones that make every other layer reachable. Here's who does what:

JobWhich spec
Package know-how an agent can loadSkills
Connect an agent to tools and dataMCP
Find what agents exist (catalog & search)ARD
Call an agent directly (the method surface)A2A
Prove who an agent is across organizationsANS / DIDs

Live presence: know which agents are reachable right nowAgentMesh
Publish and subscribe to events (one to many)AgentMesh

Skills teach. MCP equips. ARD finds. A2A talks. AgentMesh connects.

The alignment is deliberate, not incidental: AgentMesh's task model is wire-compatible with A2A, its manifests publish as ARD catalog entries, and its identities express as DIDs. The spec states exactly what it leverages, cedes, and owns.

the mesh at work

One ask, many answers

Connection changes what a request can be. A regular API is one to one: you call a service, it answers. On the mesh an agent can ask everyone at once: emit a call, and every capable agent that subscribes can reply. Collect the answers, pick a winner.

the unit of the mesh

Everything is a signed envelope

There is no client and server, no address to look up, no API to build per pair of agents. Agents speak one message format. Each envelope is signed by the sending agent and verified on receipt, so the from field is always trustworthy.

type   request
from   agent.k3f9…
to     agent.9ab1…
skill  summarize.text
trace  7c1a…·span 0f
Ed25519 signature verified · node vouches for agent
  • Addressable. An agent's public key is its address. No registry lookup to be reached.
  • Vouched. A node holds one connection and signs an attestation for each agent it hosts.
  • Traceable. Trace context rides in the envelope, so a request's path reconstructs end to end.

the protocol surface

Six operations, three pairs

The whole protocol is six operations. They are easiest to hold as three complementary pairs: one to join and find, one to call and answer, one to broadcast and listen.

pair 01 · presence

Join & find

register

Publish a signed manifest of what your agent can do.

discover

Query for agents by capability, skill, or who is online now.

pair 02 · direct

Call & answer

request

Ask an agent to act. Quick work replies inline; long work becomes a Task.

respond

Return the result, bare or as streamed Task updates while it runs.

pair 03 · events

Broadcast & listen

emit

Publish a signed event to a topic, one to many, fire and forget.

subscribe

Listen with wildcards (mesh.event.task.>) and react as things happen.

who is who

Identity chains, so trust does too

Every message traces to a root. An operator vouches for a node, a node vouches for the agents it hosts, and each agent signs its own envelopes. One machine can host many agents on a single connection without a credential per agent.
