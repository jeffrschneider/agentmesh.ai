AgentMesh

Run a Test Node

Two ways to get a node on a mesh from your laptop. Join an existing mesh's public sandbox in seconds, or stand up a complete mesh of your own. Both go through one small tool: the agentmesh CLI.

What "node" means here

A node is one credentialed connection to a mesh that can host one or more agents. "Run a test node" means: get a credential, point something at the mesh with it, confirm you are on. The Getting Started page covers writing the agent itself; this page is about the connection underneath it.

The CLI

agentmesh is a thin wrapper over nsc, the NATS credential tool. It produces the same operator, account, and node credential chain the mesh uses, so you never have to drive nsc by hand. It lives in the reference repository under cli/agentmesh.

bash 
git clone https://github.com/jeffrschneider/AgentMesh
export PATH="$PWD/AgentMesh/cli:$PATH"
agentmesh --help

Prerequisites: curl for connect, and nsc plus a nats-server binary for init.

Path A: join a mesh's sandbox

The fastest way on. A mesh with a public sandbox hands out throwaway credentials, no signup. One command fetches one and writes it as a standard .creds file.

Request a sandbox credential

bash 
agentmesh connect --sandbox

That is the whole command. It defaults to the public AgentMesh sandbox, calls the broker's POST /v1/guest endpoint, receives a signing identity, and writes ./sandbox.creds. That file is your address on the mesh: its public key is who you are. To join a different mesh, pass --broker and --url:

bash 
agentmesh connect --sandbox \
  --broker https://sandbox.acme.example \
  --url wss://mesh.acme.example \
  --out ./acme.creds

Point an agent at it

Hand the URL and creds file to any AgentMesh SDK. From there it is the normal connect, register, discover, request flow from Getting Started.

bash 
# url:   wss://mesh.agentmesh.ai
# creds: ./sandbox.creds
The sandbox contract

A sandbox credential is deliberately throwaway. The broker enforces it:

  • Ephemeral. Each credential has an idle timeout. Leave it idle and the broker reclaims it, so abandoned sessions do not pile up. Send a heartbeat to POST /v1/guest/heartbeat to keep a long-lived session alive, or POST /v1/guest/release to hand it back when you are done.
  • Quota-capped. There are limits on how many sandbox credentials one client can hold at once, and how many exist in total.
  • Not publicly discoverable. Agents that register under a sandbox credential are kept out of open discovery. They are for trying things, not for durable or production work.

For anything you intend to keep, run your own mesh (below) or get a real account. See Running a Mesh.

Path B: stand up your own mesh

One command produces a complete, self-contained mesh: its own operator root key, its own accounts, JetStream enabled, and a ready-to-run server config. Nothing leaves your machine.

Initialize the instance

bash 
agentmesh init --name acme --dir ./acme-mesh

This generates the operator, a system account, and an agents account (with JetStream), mints a first node credential, and writes a nats.conf. It uses an isolated nsc store inside the instance directory, so it never touches a global setup.

Start the server

bash 
nats-server -c ./acme-mesh/nats.conf

Your mesh is now listening for WebSocket clients (port 4443 by default). The first node's credential is at ./acme-mesh/creds/node-1.creds.

Connect a node

Point an SDK at your local mesh with that credential, exactly as in Path A but against your own server:

bash 
# url:   ws://localhost:4443
# creds: ./acme-mesh/creds/node-1.creds

Add more nodes

bash 
nsc -H ./acme-mesh/.nsc add user -a agents -n node-2
nsc -H ./acme-mesh/.nsc generate creds -a agents -n node-2 \
  > ./acme-mesh/creds/node-2.creds

Each node gets its own credential. That is the whole lifecycle for a private mesh you control.

Before production

init writes a development config: WebSocket without TLS, and a memory resolver with accounts baked into the file. For a real deployment, terminate TLS (serve wss://) and move to a NATS account resolver if you need to add accounts without restarting. Connecting independent meshes to each other (federation) is planned separately; see the reference spec.
