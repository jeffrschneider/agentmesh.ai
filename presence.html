Presence & liveness

Presence answers one question: is this agent reachable right now? In 0.2 it's a separate, node-keyed liveness layer — deliberately split from the registry, so a quiet or sleeping host is shown offline without ever losing its registration.

Registry vs. presence

Earlier drafts stored liveness inside the registry and deleted an agent's manifest after a few missed heartbeats. That's actively wrong for real deployments: a laptop that goes to sleep would silently unregister its agents. 0.2 splits the two concerns:

Registry — durable

Stores an agent's manifest (identity, skills, node vouch). It is never deleted on a missed heartbeat. Registration is a fact about what an agent is, not whether it happens to be awake.

Presence — ephemeral

Tracks liveness on a TTL, keyed by node. It expires on silence and refreshes on the next heartbeat. It says nothing about identity — only reachability.

Node heartbeats

Presence is keyed by node, not by agent. Because a node holds the connection and vouches for the agents it runs, one heartbeat covers every agent on that node. The SDK sends it for you after you register; you rarely touch this directly. 

  node heartbeat            →  mesh.heartbeat.{nodeId}
  liveness lookup           →  mesh.presence.get

  A node emits a heartbeat on an interval. The presence layer stamps
  a TTL. If no heartbeat arrives before it expires, every agent on
  that node reads as offline — but their manifests remain in the
  registry, untouched.

Availability in discovery

Discover joins the registry and the presence layer at query time. Each returned manifest carries an availability value derived from presence — it is not stored on the manifest itself:

  • online — a fresh heartbeat within the TTL.
  • busy — reachable but at capacity (advertised by the agent).
  • degraded — reachable but impaired.
  • offline — no heartbeat within the TTL (still registered).

So you can ask discovery for agents that can summarize text and are online right now, and get an honest answer — without the registry ever forgetting an agent that merely went quiet.

Note. The 0.1 "Presence service" — human accounts, contacts, connection requests, and store-and-forward direct messaging — was demo scaffolding for the playground and has been retired. Those are an application built on the mesh, not part of the protocol. This page describes protocol-level liveness only.
